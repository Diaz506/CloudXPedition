⬜ AZ-900: Azure Fundamentals
+50 XP
🚗 Analogy — Your Driver's License for the Cloud: Think of AZ-900 as your driver's license for the cloud. Just like a driver's license proves you know the rules of the road before you start driving professionally, AZ-900 proves you understand cloud fundamentals before you start building solutions for customers.

Just as a driving test covers road signs, traffic laws, and basic vehicle operation, the AZ-900 exam covers cloud concepts, core Azure services, security basics, and pricing models. It's your official entry pass into the world of cloud professionals.

📘 Study Guide

Exam Overview

  • Format: 40–60 questions (multiple choice, drag-and-drop, scenario-based)
  • Passing score: 700 out of 1000
  • Duration: 65 minutes
  • Cost: ~$165 USD (discounts available for students)

Recommended 4-Week Study Plan

  • Week 1: Cloud concepts — IaaS, PaaS, SaaS, shared responsibility, CapEx vs OpEx
  • Week 2: Core Azure services — compute, networking, storage, databases
  • Week 3: Security, privacy, compliance, and trust — Azure AD, NSGs, Azure Policy
  • Week 4: Pricing, SLAs, and lifecycle — TCO calculator, Azure Advisor, support plans

Key Topics

  • Cloud concepts (25–30%)
  • Azure architecture and services (35–40%)
  • Azure management and governance (30–35%)

Practice Tests & Registration

  • Complete at least 3 full practice exams before scheduling
  • Aim for consistent 85%+ on practice tests before booking
  • Register at Microsoft Learn or through Pearson VUE
  • Choose online proctored or in-person testing center

⬜ Next Certification Path
+50 XP
🏍️ Analogy — Motorcycle or Commercial License? After your driver's license, do you get a motorcycle license (specialized) or a commercial license (broader)? It depends on where you want to go. Similarly, your next cloud certification should match your career direction.

Just as a motorcycle license makes you an expert rider and a commercial license lets you drive bigger rigs, choosing between AZ-104 (Azure Admin — deep Azure specialization) and AWS SAA (Solutions Architect — broader multi-cloud appeal) shapes your professional path. Neither is wrong — it depends on your destination.

🗺️ Certification Roadmap

AZ-104: Azure Administrator

  • Focus: Managing Azure identities, governance, storage, compute, and virtual networks
  • Best for: Roles in Azure-centric organizations, Microsoft partner companies
  • Study timeline: 8–10 weeks (assuming AZ-900 foundation)
  • Career path: Azure Admin → Azure Solutions Architect (AZ-305) → Azure DevOps (AZ-400)

AWS Solutions Architect Associate (SAA-C03)

  • Focus: Designing distributed systems, cost optimization, resilient architectures on AWS
  • Best for: Broader market appeal, startups, AWS-heavy shops
  • Study timeline: 10–12 weeks
  • Career path: SAA → SA Professional → DevOps Professional or Specialty certs

How to Choose

  • Check your employer: What cloud does your company or target employer use?
  • Check job postings: Which cert appears more in roles you want?
  • Multi-cloud strategy: Many pre-sales professionals eventually get both
  • Tip: Start with the cloud your employer uses, then branch out

⬜ Building Your Portfolio
+50 XP
👨‍🍳 Analogy — A Chef's Tasting Menu: You don't just say you can cook — you SHOW your best dishes. A chef's tasting menu lets diners experience your range, creativity, and skill. Your cloud portfolio is your tasting menu for employers: curated projects that prove you can actually do the work.

Just as a tasting menu features a starter, main, and dessert to show range, your portfolio should showcase different types of architectures — a web app, a serverless solution, and a hybrid setup — to demonstrate versatility. Each "dish" needs a clear recipe (documentation) so others can follow your thinking.

🍽️ Portfolio Guide

Sample Architectures to Build

  • 3-Tier Web Application: Frontend (App Service / Static Web App) → API (Azure Functions / App Service) → Database (Azure SQL / Cosmos DB). Include load balancing, CDN, and monitoring.
  • Serverless API: API Management → Azure Functions → Cosmos DB with Event Grid triggers. Show event-driven patterns and auto-scaling.
  • Hybrid Network: On-premises datacenter connected via VPN Gateway or ExpressRoute to Azure VNets. Include DNS resolution, NSGs, and Azure Arc for hybrid management.

Demo Environments

  • Maintain at least one live demo you can show in interviews
  • Use Azure free tier and teardown scripts to manage costs
  • Create ARM/Bicep templates for quick reprovisioning

GitHub Portfolio Setup

  • Create a dedicated GitHub repository for each project
  • Include: README with architecture diagram, problem statement, solution design, deployment instructions
  • Add a portfolio README profile (github.com/username/username) linking all projects

Writing Case Studies

  • Format: Problem → Requirements → Solution → Architecture Diagram → Results
  • Use real-sounding scenarios even if the project is personal
  • Include cost estimates and trade-off decisions

🛠️ Practical Exercise

Create 3 reference architectures, document each with diagrams (use draw.io or the Azure architecture icons), and present them to a friend or colleague for feedback. Practice explaining your design decisions out loud.

🎭 Analogy — Dress Rehearsal Before Opening Night: No actor walks on stage without rehearsals. A dress rehearsal is where you practice your performance — lines, timing, blocking — until it feels natural, not rehearsed. Mock pre-sales scenarios are your dress rehearsals: practice until your cloud pitches feel effortless.

Just as actors rehearse with different audiences and unexpected situations (a prop breaks, an actor forgets a line), you need to practice with different customer types and curveball questions. The goal isn't to memorize scripts — it's to internalize the concepts so deeply that you can adapt on the fly.

🎤 Practice Scenarios

How to Practice

  • Read each scenario, then speak your response out loud (not just in your head)
  • Time yourself — aim for 3–5 minute responses per scenario
  • Record yourself and play it back to catch filler words and weak points
  • Practice with a partner who asks follow-up questions

Scenario 1: Healthcare Company — HIPAA Compliance Concerns 🏥

"We're a healthcare provider with 50 clinics. We want to move to the cloud but our CISO is concerned about HIPAA compliance. Can Azure handle our needs?"

  • Key talking points:
  • Azure offers a HIPAA BAA (Business Associate Agreement) as standard
  • Azure has 90+ compliance certifications, including HITRUST
  • Azure Confidential Computing for sensitive workloads
  • Reference architecture: Azure Health Data Services
  • Customer story: Highlight healthcare customers already on Azure
  • Common objection: "But our data can't leave our region" → Azure has data residency guarantees and sovereign cloud options

Scenario 2: Startup — Scale Fast on a Tight Budget 🚀

"We're a 10-person startup with $5K/month cloud budget. We need to launch our SaaS product in 3 months and handle unpredictable traffic spikes. Why Azure over AWS?"

  • Key talking points:
  • Azure free tier + $200 startup credits (BizSpark/Founders Hub: up to $150K credits)
  • Serverless-first approach: Azure Functions + Cosmos DB serverless = pay only for usage
  • Azure Static Web Apps for frontend — free tier with built-in CI/CD
  • Auto-scaling without managing infrastructure
  • Competitive angle vs AWS: Simpler pricing model, better dev tooling integration (VS Code, GitHub Actions)
  • Common objection: "AWS has more market share" → Market share ≠ best fit. Azure's PaaS offerings reduce your ops burden so your small team can focus on product.

Scenario 3: Enterprise — Legacy Migration Fear 🏭

"We're a manufacturing company with 300+ legacy Windows apps, some running on Windows Server 2012. Our team is afraid migration will break everything. Convince us."

  • Key talking points:
  • Azure Migrate: automated discovery and assessment of all 300+ apps
  • Lift-and-shift first (Azure VMs), then modernize incrementally — no big-bang migration
  • Extended Security Updates for Windows Server 2012 free on Azure (saves licensing costs)
  • Azure Arc for hybrid management — manage on-prem and cloud from one pane
  • Phased migration: pilot with 10 non-critical apps, prove the pattern, then scale
  • Common objection: "We can't afford downtime" → Azure Site Recovery enables near-zero downtime migration with continuous replication

🐉 Final Boss Challenge: The Complete Pre-Sales Pitch

This is it — the FINAL challenge of the entire Cloud Pre-Sales Learning Path. Give it everything you've got!

🏭 The Scenario: You've been invited to present to the C-suite of TitanForge Manufacturing, a large manufacturing company with 5,000 employees across 12 factories worldwide. They are evaluating Azure, AWS, and GCP and you're the Azure representative. Here's what you're facing:
  • 🔒 Security: They suffered a data breach 6 months ago — the CISO is skeptical of cloud security
  • 💰 Costs: The CFO has a tight budget and wants clear ROI within 12 months
  • 📋 Compliance: ISO 27001 is required for their European operations
  • 🖥️ Migration: 300+ legacy apps, many on Windows Server 2012 and 2016

Your mission: Answer each part of the pitch below. Type your responses, then reveal the sample answer to compare. Every part you complete earns you progress toward defeating this boss!

Part 1: Opening Statement

How do you open the meeting? You have 60 seconds to capture the room's attention and set the tone.

Sample: "Thank you for having me. I know TitanForge has been through a challenging year with the security incident, and I'm not here to minimize that. I'm here because Microsoft Azure has helped manufacturers just like you — companies like BMW, Bosch, and Johnson Controls — not only recover from security challenges but come out stronger. Today, I'll show you a practical, phased approach that addresses your security concerns first, delivers measurable ROI within 12 months, and migrates your 300+ apps without disrupting production."

Part 2: Addressing Security Concerns

The CISO asks: "After our breach, how can you guarantee cloud is safer than keeping everything on-premises?"

Sample: "I won't guarantee it — no one honestly can. But here's what I can show you: Microsoft invests over $4 billion annually in cybersecurity and employs 8,500+ security professionals. Azure's Zero Trust architecture, Microsoft Defender for Cloud, and Azure Sentinel provide threat detection across your entire estate — something that's extremely expensive to replicate on-prem. We can also implement Azure Confidential Computing so even Microsoft can't access your data in use. Most importantly, I'd recommend starting with a security assessment using Microsoft Defender to identify your current vulnerabilities — at no additional cost."

Part 3: Presenting ROI

The CFO says: "Show me the money. Why should I spend millions migrating when our current servers still work?"

Sample: "Great question — let's look at the numbers. First, your Windows Server 2012 machines are out of support — you're paying for Extended Security Updates or running unpatched. Azure gives you those ESUs free, saving approximately $X per server per year. Second, Azure Reserved Instances can cut compute costs by up to 72%. Third, the TCO Calculator shows most manufacturers save 30-40% over 3 years. But here's the real ROI: Azure IoT and AI services can optimize your factory floor — predictive maintenance alone typically saves manufacturers 12-18% in maintenance costs. I'd like to run a TCO analysis with your actual numbers — can your IT team share current infrastructure spend?"

Part 4: Compliance Approach

The legal team asks: "We need ISO 27001 for our European factories. How does Azure handle this?"

Sample: "Azure is already ISO 27001 certified — it's one of our 90+ compliance certifications. But your certification covers your entire organization, not just the cloud provider. Here's how we help: Azure Policy enforces compliance rules automatically — no human error. Microsoft Purview tracks your data across all systems. Azure Blueprints deploy pre-configured environments that meet ISO 27001 controls out of the box. For your European operations specifically, we have data centers in EU regions with data residency guarantees, fully compliant with GDPR. I can share our ISO 27001 audit reports under NDA today."

Part 5: Migration Strategy

The CTO asks: "We have 300+ apps, some 15 years old. How do you migrate without breaking everything?"

Sample: "We don't. Not all at once. Here's the proven approach: Phase 1 — Azure Migrate discovers and assesses all 300+ apps automatically, categorizing them into lift-and-shift, re-platform, and re-architect. Phase 2 — We start with 10–15 non-critical apps as a pilot, proving the process and training your team. Phase 3 — We use Azure Site Recovery for near-zero downtime migration of the next batch. Phase 4 — Critical apps are migrated with parallel run (old and new running simultaneously) until you're confident. For your legacy Windows Server 2012 apps, many can lift-and-shift to Azure VMs unchanged — and Azure Arc lets you manage everything from one dashboard during the transition."

Part 6: Competitive Positioning vs AWS & GCP

The CEO asks: "AWS is the market leader and Google has the best AI. Why should we pick Azure?"

Sample: "Great question — all three are excellent platforms. Here's why Azure is the best fit for TitanForge specifically: First, your 300+ apps are Windows-based — Azure has the deepest Windows integration and free ESUs. AWS would charge extra. Second, your teams already use Microsoft 365, Active Directory, and Teams — Azure integrates natively, reducing complexity and training costs. Third, Azure's hybrid story (Azure Arc, Azure Stack HCI) is strongest for manufacturers who need to keep some workloads on-premises near factory equipment. On AI — Azure OpenAI Service gives you the same GPT models as Google's competition, fully integrated into your Azure environment. I'd never say 'don't evaluate others' — but for your specific situation, Azure aligns best."

Part 7: Closing Statement

You have 30 seconds for your closing. Make it count.

Sample: "Here's what I'd like to propose: a no-cost, two-week Azure Proof of Concept. We'll migrate 5 of your non-critical apps, run a full security assessment, and deliver a TCO analysis with real numbers. No commitment, no risk — just evidence. If the results don't speak for themselves, you've lost nothing. But based on what I've seen with manufacturers like you, I'm confident you'll see why Azure is the right platform for TitanForge's next chapter. Can we schedule the kickoff?"
