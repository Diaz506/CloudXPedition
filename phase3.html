🏗️ IaaS, PaaS, SaaS
The three flavors of cloud service
+50 XP
💡 Think of it this way:
🏚️ IaaS = An unfurnished apartment — you get the walls and plumbing, but you bring your own furniture, appliances, and decor.
🛋️ PaaS = A furnished apartment — move in and start living. The basics are handled for you.
🏨 SaaS = A hotel room — everything is done for you. Just show up and use it.

IaaS (Infrastructure as a Service) gives you virtual machines, storage, and networking — you manage the OS and everything above it. Examples: Azure VMs, AWS EC2. PaaS (Platform as a Service) gives you a ready-made platform to deploy your apps — no managing servers. Examples: Azure App Service, AWS Elastic Beanstalk. SaaS (Software as a Service) is fully managed software you just use. Examples: Microsoft 365, Salesforce, Gmail. The shared responsibility model shifts: with IaaS you manage most things; with SaaS the provider handles almost everything — like the difference between being a landlord vs a hotel guest.

🔵 Azure Fundamentals (AZ-900)
Your home base in the cloud
+50 XP
💡 Think of it this way: Azure is like a massive shopping mall with hundreds of stores (services). Resource Groups are your shopping bags — they keep related purchases together. Subscriptions are your credit cards — they determine who pays. Regions are different mall locations around the world — you pick the one closest to your customers.

Azure Portal is the dashboard where you manage everything — think of it as the mall directory. Resource Groups are logical containers: put all resources for a project together so you can manage, monitor, and delete them as a unit. Regions are physical datacenter locations worldwide — choosing the right region means lower latency for your users. Subscriptions are billing boundaries — departments or projects can have separate subscriptions for cost tracking. Core services include Compute (VMs, App Service), Storage (Blob, Files), Networking (VNet, Load Balancer), and Databases (SQL, Cosmos DB).

🟠 AWS Cloud Practitioner Basics
Know the competition
+50 XP
💡 Think of it this way: If Azure is a shopping mall you know well, AWS is the competing mall across town. The stores have different names but sell similar things. Knowing both makes you a better shopper (and a better pre-sales consultant — customers use both!).

EC2 = AWS's virtual machines (like Azure VMs). S3 = Object storage (like Azure Blob Storage) — where you store files, backups, images. IAM = Identity and Access Management (like Azure AD/Entra ID for AWS). Regions & Availability Zones = Same concept as Azure regions, data centers spread globally. VPC = Virtual Private Cloud (like Azure VNet) — your private network in AWS. You don't need to be an AWS expert, but being conversational about it is essential in pre-sales — many customers use multi-cloud or are migrating between providers.

🤝 Shared Responsibility Model
Who's responsible for what?
+50 XP
💡 Think of it this way: Renting an apartment — the landlord maintains the building structure (foundation, roof, plumbing), but you're responsible for locking your door, not leaving the stove on, and keeping your stuff safe. The cloud works the same way: the provider secures the infrastructure, you secure your data and access.

The shared responsibility model defines who handles what:
With IaaS: The provider manages physical security, networking, and hardware. You manage the OS, applications, data, and access control. (Like renting an unfurnished apartment — the building is maintained, but everything inside is on you.)
With PaaS: The provider also handles the OS and runtime. You manage the application code and data. (Furnished apartment — less for you to worry about.)
With SaaS: The provider handles almost everything. You manage your data, accounts, and access permissions. (Hotel — just don't lose your room key.)
Understanding this model is critical in pre-sales conversations — customers often assume "cloud = someone else's problem." Your job is to clarify what they still own.

🔄 Hybrid Cloud & Migration Strategies
Getting to the cloud — the right way
+50 XP
💡 Think of it this way: Keeping valuables in a home safe (on-prem) but using a bank safe deposit box (cloud) for extra security. Most people don't move everything to the bank overnight — they gradually shift things based on value and convenience. That's hybrid cloud.

Migration strategies (the "6 R's"):
Lift-and-Shift (Rehost): Move as-is to the cloud. Fast but doesn't optimize for cloud benefits. Like moving your furniture to a new apartment without changing anything.
Re-platform: Small tweaks during migration. Like rearranging furniture in the new place for a better layout.
Re-architect: Redesign the application to be cloud-native. Most effort, but greatest long-term benefits. Like gutting and renovating the apartment.
Azure Arc extends Azure management to on-prem and multi-cloud resources — manage everything from one pane of glass.
Reality check: Nobody goes 100% cloud overnight. Migration is a journey, not a switch.

💰 Cloud Economics & FinOps
The money side of cloud
+50 XP
💡 Think of it this way: CapEx (Capital Expenditure) is buying a car — big upfront cost, you own it, it depreciates. OpEx (Operational Expenditure) is using Uber — pay per ride, no maintenance, scale up or down as needed. Cloud shifts IT from buying cars to using Uber.

Pay-as-you-go: Only pay for what you use, like a utility bill. No massive upfront investment. Reserved Instances: Commit to 1-3 years for significant discounts (up to 72% off). Like signing a long-term lease for a lower monthly rate. TCO Calculators: Tools that compare the total cost of running workloads on-prem vs cloud — including hidden costs like power, cooling, staff, and real estate. Cost Management Tools: Azure Cost Management, AWS Cost Explorer — dashboards that track spending and set budget alerts before you get a surprise bill. FinOps is the practice of bringing financial accountability to cloud spending — making teams responsible for what they consume, just like a department budget.

🧠 Knowledge Check

A startup wants to deploy a web application. They don't want to manage servers or operating systems — just deploy their code and let it run. They expect unpredictable traffic spikes. Which cloud model fits best?

A) IaaS — spin up virtual machines and install the web server yourself
B) PaaS — deploy code to a managed platform like Azure App Service that auto-scales
C) SaaS — use an existing website builder like Wix
✅ Correct! PaaS is perfect here — the startup deploys their custom code without managing infrastructure, and the platform handles scaling automatically during traffic spikes. IaaS would require managing servers; SaaS wouldn't allow custom application code.
❌ Not quite. The key requirements are: custom code deployment + no server management + auto-scaling. PaaS (like Azure App Service) is designed exactly for this — it handles the infrastructure while the team focuses on code.

🏆 Phase Boss Challenge: "The Cloud Strategy Advisor"

Their servers are 7 years old and approaching end-of-life. They have strict data regulations requiring some data to stay in-country. The CFO is concerned about the large capital expense of replacing all servers."

Part 1: What migration approach makes sense for the aging ERP system that's deeply customized?

A) Re-architect it immediately as a cloud-native application
B) Lift-and-shift to Azure VMs first, then modernize gradually
C) Replace it with a SaaS ERP immediately

Part 2: How should you address the CFO's concern about capital expenses?

A) Tell them cloud is always cheaper — show only the pay-as-you-go prices
B) Use the Azure TCO calculator to compare total costs, showing the shift from CapEx to OpEx and including hidden on-prem costs
C) Suggest they just buy new on-prem servers since they already have the infrastructure

Part 3: Given the data residency regulations, what cloud strategy best fits?

A) Full public cloud — just pick a region in their country
B) Stay fully on-premises — regulations make cloud impossible
C) Hybrid cloud — regulated data stays on-prem or in a sovereign cloud region, while other workloads move to Azure
