🖥️ Windows Server Basics
The backbone of enterprise IT
+50 XP
💡 Think of it this way: A server OS is like a restaurant kitchen — it's designed to serve many people efficiently, not just one person sitting at home. Windows Server is the industrial kitchen behind every big enterprise, running services that hundreds or thousands of employees depend on.

Unlike your home Windows that's designed for one person, Windows Server is built to handle many users simultaneously and run critical services. Server Roles are like job titles: AD DS (manages users and permissions), DNS Server (translates names to addresses), DHCP Server (assigns IP addresses), and File Server (shared storage for the company). One physical server can wear many hats — or you can spread roles across multiple servers for reliability.

📂 Active Directory
The company's employee directory + badge system
+50 XP
💡 Think of it this way: Active Directory is the company's employee directory combined with the badge system — it knows who works here, what teams they're on, and which rooms they can access. When you swipe your badge to enter a room, AD is the system checking if you're allowed in.

Users are individual employee accounts. Groups bundle users together — "Everyone in Marketing" or "All Managers." Organizational Units (OUs) are like departments on an org chart — they organize users and computers into logical containers. Group Policy (GPO) is like company-wide rules enforced automatically — "All computers must have a screen lock after 5 minutes" or "Nobody can install unapproved software." Set it once, and it applies to thousands of machines instantly.

🔐 Azure AD / Entra ID
Identity in the cloud era
+50 XP
💡 Think of it this way: SSO (Single Sign-On) is one master key that opens every door in the building — log in once and access all your apps. MFA (Multi-Factor Authentication) is like a bank vault that needs BOTH a key AND your fingerprint — even if someone steals your password, they still can't get in.

Microsoft Entra ID (formerly Azure AD) is Active Directory's cloud-native sibling. While traditional AD manages your office network, Entra ID manages identity for cloud apps like Microsoft 365, Salesforce, and thousands more. Conditional Access is like a smart bouncer: "You can enter if you're on a company device, from a trusted location, and passed MFA — otherwise, extra verification needed." Hybrid Identity syncs your on-prem AD with Entra ID so users have one identity everywhere — office and cloud.

🐧 Basic Linux
The other side of the server world
+50 XP
💡 Think of it this way: Learning Linux after Windows is like learning to drive a manual car after only driving automatic — different controls, same fundamental concept. The steering wheel is still there, but now you have a clutch pedal (the command line) that gives you more control.

Linux runs the majority of cloud servers, so basic familiarity is essential. The command line is your primary tool: ls (list files), cd (change directory), pwd (where am I?), cat (read a file). File permissions control who can read, write, or execute files — think of it as "who has the key to which filing cabinet." Package managers (apt on Ubuntu, yum on Red Hat) are like app stores for the command line — they install, update, and remove software.

⚡ Basic Scripting & Automation
Talk directly to the computer
+50 XP
💡 Think of it this way: Scripting is talking directly to the computer in its language instead of clicking through menus. Imagine telling an assistant "sort these 10,000 files by date" — doing it manually would take days. A script does it in seconds.

PowerShell is Microsoft's powerful scripting language — it can manage Windows servers, Azure resources, Microsoft 365, and more. It speaks in "verb-noun" commands: Get-Process, Set-Location, New-Item. Azure CLI is a command-line tool specifically for managing Azure resources — create VMs, deploy apps, manage storage, all from the terminal. Automation matters at scale: clicking through a GUI to set up one server is fine, but when you need to configure 100 servers identically, scripting is the only sane option.

🏠 On-Premises vs Cloud
Own it or rent it?
+50 XP
💡 Think of it this way: On-premises is like owning your house — you have full control, but YOU'RE the one fixing the plumbing at 3 AM. Cloud is like renting an apartment — the landlord handles maintenance, but you follow their rules and pay monthly.

On-premises pros: Full control, data stays in your building, no monthly cloud bills (after initial investment), works without internet. On-premises cons: Huge upfront cost, you maintain everything, scaling means buying more hardware, disaster recovery is your problem. Cloud pros: Pay-as-you-go, scales instantly, provider handles hardware, global availability, built-in backup options. Cloud cons: Ongoing costs, internet dependency, less physical control, potential compliance concerns. The reality: Most companies go hybrid — some workloads on-prem, some in the cloud. It's rarely all-or-nothing.

🧠 Knowledge Check

A company with 200 employees wants to allow staff to access email, file shares, and a CRM app with a single login. Several employees work remotely. What should the IT team implement?

A) Give every employee separate passwords for each application for maximum security
B) Implement Azure AD / Entra ID with SSO and MFA, synced with on-prem Active Directory
C) Set up a VPN and require employees to be on-site for all application access
✅ Correct! Hybrid identity with SSO gives users one login for everything, MFA adds security for remote access, and syncing with on-prem AD maintains existing infrastructure. This is the modern enterprise approach.
❌ Not quite. With 200 employees and remote workers, you need a centralized identity solution. Azure AD/Entra ID with SSO and MFA is the industry standard — it gives convenience (one login) with security (MFA) across all locations.
