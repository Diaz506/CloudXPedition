⬜ 1. Cloud Compute Services +50 XP
▶
🏠 Analogy: Think of renting an apartment vs. booking a hotel room. Virtual Machines are like long-term apartment leases — you get full control, but you handle the furniture and repairs. App Services are like hotel stays — someone else handles the maintenance; you just show up and work.

Cloud compute is the engine behind every workload. Azure VMs (or AWS EC2) give you raw virtual hardware — full OS access, custom installs, total control. Azure App Service (or AWS Elastic Beanstalk) abstracts the infrastructure so you deploy code, not servers. Choose VMs when you need deep customization; choose managed services when you want speed and simplicity.

⬜ 2. Containers & Kubernetes +50 XP
▶
🚢 Analogy: Imagine shipping containers on a cargo ship. Each container is standardized, portable, and isolated — it doesn't matter what's inside; it fits on any ship. That's Docker. Now picture a harbor master who decides which containers go on which ship, replaces damaged ones, and balances the load — that's Kubernetes.

Containers package your app and all its dependencies into a single portable unit. Unlike VMs, they share the host OS kernel, making them lighter and faster to start. Docker is the standard for building containers. AKS (Azure Kubernetes Service) and EKS (AWS) orchestrate thousands of containers — handling scaling, healing, and load balancing automatically.

⬜ 3. Cloud Networking +50 XP
▶
🛣️ Analogy: A VPN is like a private tunnel through a public highway — your data travels securely even over the open internet. A load balancer is like a restaurant host who distributes customers evenly across available tables so no single waiter is overwhelmed. ExpressRoute is a private road built directly from your office to the cloud — no public traffic at all.

Cloud networking connects everything. Virtual Networks (VNets in Azure, VPCs in AWS) create isolated network spaces. NSGs act as firewalls controlling traffic. Load Balancers distribute requests across healthy instances. VPN Gateway creates encrypted tunnels to on-premises, while ExpressRoute/Direct Connect provides dedicated private connectivity for mission-critical workloads.

⬜ 4. Cloud Storage Services +50 XP
▶
🏭 Analogy: Cloud storage is like a massive warehouse. You pay based on how much space you use and how often you access your stuff. Hot tier is the front shelf — instant access, higher rent. Cool tier is the back room — cheaper, but takes a moment to grab. Archive tier is off-site deep storage — very cheap, but retrieval takes hours.

Azure Blob Storage (or AWS S3) stores unstructured data — images, backups, logs — at massive scale. Managed Disks attach directly to VMs like virtual hard drives. Azure Files offers SMB file shares in the cloud. Understanding access tiers (hot, cool, archive) is key to optimizing storage costs for customers.

⬜ 5. Cloud Database Services +50 XP
▶
📚 Analogy: Think of a library. SQL databases are like a well-organized card catalog — every book has a fixed place, and you search by exact categories. NoSQL databases are like a free-form notebook — each page can look completely different, which gives you flexibility but less structure.

Azure SQL and AWS RDS provide managed relational databases — patching, backups, and HA built in. Cosmos DB and DynamoDB handle NoSQL workloads with global distribution and flexible schemas. The key pre-sales question: does the customer's data have a fixed structure (→ SQL) or does it vary wildly (→ NoSQL)? Managed services eliminate the "database admin at 3 AM" problem.

⬜ 6. Backup & Disaster Recovery +50 XP
▶
📓 Analogy: Imagine your notebook is destroyed. RPO (Recovery Point Objective) answers: "When was the last photocopy?" — how much work can you afford to lose? RTO (Recovery Time Objective) answers: "How fast can you get a new notebook and rewrite?" — how long can the business be down?

Azure Backup provides automated, policy-driven backups for VMs, databases, and files. Azure Site Recovery (ASR) replicates entire workloads to a secondary region for disaster recovery. Cross-region replication ensures data survives even regional outages. When talking to customers, RPO and RTO are the two magic numbers that shape every DR strategy and its cost.

⬜ 7. High Availability & SLAs +50 XP
▶
🍽️ Analogy: Think of a restaurant chain. If one kitchen catches fire, the other locations keep serving customers. That's high availability — redundancy across locations so a single failure doesn't bring everything down. The SLA is the restaurant's promise: "We guarantee 99.99% of your orders will be served on time."

Availability Zones are physically separate datacenters within an Azure region — your app runs in multiple zones so one datacenter failure doesn't cause downtime. SLA tiers matter: 99.9% allows ~8.7 hours of downtime/year; 99.99% allows only ~52 minutes/year. In pre-sales, helping customers understand the cost vs. uptime tradeoff is a huge value-add.

⬜ 8. Monitoring & Observability +50 XP
▶
🚗 Analogy: Think of the dashboard instruments in a car. You have a speedometer, fuel gauge, and warning lights. A check-engine light comes on before you break down — that's monitoring. Observability goes deeper: it's like having a mechanic's diagnostic tool that tells you exactly which part is failing and why.

Azure Monitor is the central hub for metrics and logs. Log Analytics lets you query logs with KQL. Application Insights tracks app performance — response times, failures, dependencies. Alerts notify your team before users even notice problems. For pre-sales, monitoring is how you prove the cloud is working and catch issues before they become outages.

⬜ 9. Cloud Security & Zero Trust +50 XP
▶
✈️ Analogy: Think of airport security. Even after you pass the main checkpoint and you're inside the terminal, you still get checked at every gate before boarding. That's Zero Trust — never assume someone is safe just because they're "inside" the network. Verify explicitly, every time.

Zero Trust follows three principles: verify explicitly, use least-privilege access, and assume breach. Azure Key Vault stores secrets and certificates securely. IAM policies and RBAC control who can do what. Conditional Access adds context-aware rules (e.g., block sign-ins from risky locations). In pre-sales, security is never optional — it's the foundation of every cloud conversation.
